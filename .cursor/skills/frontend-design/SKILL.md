# Frontend Design

## Design Principles
- Ground it in the subject: Design should reflect the content's nature
- Restraint: Less is more — avoid decorative elements that don't serve function
- Self-critique: Build, step back, critique, iterate

## Process
1. Brainstorm: Understand the user's needs and content
2. Explore: Look at references and inspiration
3. Plan: Wireframe the layout
4. Critique: Review the plan before building
5. Build: Implement with clean, semantic HTML + CSS
6. Critique again: Step back and refine

## Key Guidelines
- Use modern CSS (Grid, Flexbox, custom properties)
- Prefer system font stacks for performance
- Ensure responsive design (mobile-first)
- Accessible: semantic HTML, ARIA where needed, sufficient contrast
- Performant: minimize JS, lazy-load images, use CSS animations

## More on Writing in Design
- Typography is the foundation of web design
- Use a clear type scale (e.g., 1.25 ratio)
- Line height: 1.5-1.6 for body text
- Maximum line length: 45-75 characters
